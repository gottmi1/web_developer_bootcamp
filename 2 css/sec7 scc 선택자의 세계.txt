*{
  전체선택
}

div{
  지정된 타입의 모든 요소
}

div, article{
  여러가지 요소 지정 가능
}

#id {
  가장 우선수위가 높다
}

.class {
  id 다음
}

li a {
  자손 선택자
}

h1 + p {
  인접 선택자, 결합자 h1태그 바로 다음에 오는 모든 형제태그인 p를 선택함
}

div > li {
  직계 선택자
}

input[type="text] {
  속성 선택자. 해당 속성인(예를 들면 type이 text인 input만) 태그만을 선택한다.
}

section.post {
  post클래스를 가진 요소가 여러곳에 걸쳐 있을 때 post클래스를 가진 section만 선택한다.
}


가상클래스

선택자 끝에 붙여 상태를 특정하는 키워드이다.
:hover :nth-child이딴건 아니까 모르는것만 적음

:active 클릭하고 누르고있으면 적용되는 가상클래스
:nth-of-type(5n) 매 5번째 요소 선택

가상요소

::first-letter 
해당 문자열의 맨 첫글자에만 적용
::first-line
해당 단락의 첫 줄에만 적용
::selection
드래그 했을 때


케스케이드

적용된 스타일의 순서 : 더 아래에 오는놈이 적용됨
하나 이상의 스타일 시트가 있을 때도 아래에 있는놈이 위에 있는 스타일시트와 중복된 스타일을 사용하고 있다면 덮어씌워진다.

우선순위

서로 상반되는 스타일이 있을경우, 더 구체적인 선택자를 우선적용한다.

개발자 도구에서 스타일에 취소선이 쳐져있는 style들은 순서가 밀렸거나 특이도가 떨어졌기 때문에 밀려난 애들이다

명시도

inline 스타일(안 쓰는게 좋다) > id > class > 일반선택
!pmoprtant(안 쓰는게 좋다) 명시도 99999999로 올림


css의 상속

하위 항목에 따로 설정하지 않으면 상위 항목의 스타일을 물려받는다.
input이나 button등 자동으로 상속하지 않는 요소들도 있다.(모든 요소들이 자동으로 상속되지는 않는다.)
이런 애들은 inherit로 상속시킬 수 있음.
